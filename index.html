What is GSAP?

GreenSock Animation Platform

http://greensock.com/gsap

GSAP is a powerful javascript animation library. The creators have emphasized both the power of GSAP and the performance, even on mobile devices.

Why use GSAP instead of CSS?

Well, you wouldn't want to necessarily. For many sites and situations, CSS can handle all the animation you want with little or no downsides, without including another library.

GSAP does do some things that CSS doesn't. It supports various functions that CSS can do, but is compatible back to IE6, which CSS animations and transitions are not.

Unless these types of animations were absolutely crucial to the user for usability purposes, it is probably not necessary for older browsers to have them and thus CSS would most likely be sufficient.

Attempting a complex animation with CSS vs GSAP

See the Pen Jobbie by Caitlin Bailey (@ayashi) on CodePen.

I tried to make this animation with CSS animation first before I threw in the towel. I was experiencing very jerky animation that looked terrible and was very time consuming to adjust.

I couldn't find any good information on animating along a path (except for some SVG related items which, especially at the time, had poor browser support). Also if you notice the image does not look crisp at all which may be related to how the browser is handling the transforms.

GSAP allowed me to do the exact animation I wanted extremely easily. Editing the path was simple (though the path I ended up with was a little different from the one used in the codepen) and I got nice smooth animation and a crisp image.
